- Presentation/UI Hábitos: nuevo `HabitsViewModel` + `HabitsUiState`, pantalla `HabitsScreen` con anillo de progreso simple, textos suaves de avance y lista de 3 intenciones con CTA para marcar/desmarcar.

## 2026-04-07
- Hábitos: añadida sincronización Firestore local-first en `users/{uid}/habits/current` con snapshot único (`todayDateIso`, intenciones seleccionadas/completadas, progreso, ciclos, `updatedAtEpochMillis`).
- Data: nuevo `SyncHabitsRepository` siguiendo patrón `LocalStore + RemoteStore + SyncEngine`, con pull corto best-effort en lectura/operaciones sensibles y push asíncrono tras mutaciones.
- Data local: `SettingsHabitsRepository` ahora persiste/expone snapshot completo + `updatedAtEpochMillis` para resolver conflictos por last-write-wins entre dispositivos.
- DI: `HabitsRepository` pasa a resolverse con `SyncHabitsRepository`, manteniendo `SettingsHabitsRepository` como base local/fallback.
- Docs Firestore actualizadas (schema, rules, examples) para reflejar la nueva colección de hábitos sincronizados.

Impacto arquitectónico:
- Sin cambios de módulos ni UX. Se extiende patrón de sync existente a la feature Hábitos.

Pendientes:
- Mantener MVP sin histórico diario ni listeners realtime (fuera de alcance actual).

## 2026-04-09
- i18n foundation (subfase infraestructura): añadidos `AppLanguage` + `AppLanguageRepository` en domain con lista estable de idiomas soportados (`es`, `en`, `pt`, `ru`, `fr`, `it`, `de`) y fallback explícito a español.
- Data local-first: nuevo `SettingsAppLanguageRepository` usando `multiplatform-settings` para persistencia local de idioma (`bwitch_language/current_language_code`).
- Resolución inicial de idioma: prioridad **guardado BWitch > locale del sistema soportado > español**, encapsulada en el repositorio.
- KMP locale hook: nuevo `SystemLanguageCodeProvider` expect/actual (Android/iOS) para leer idioma del sistema sin acoplar commonMain a APIs de plataforma.
- Estado global: nuevo `AppLanguageViewModel` en presentation con estado observable + acción de cambio de idioma.
- Integración mínima UI: selector reutilizable `LanguageSelectorSection` conectado en onboarding y ajustes; sin migración masiva de textos aún.
- Resources strategy: creado primer `composeResources/values/strings.xml` para el bloque de idioma y dejar trazado el camino de migración gradual a resources.

Impacto arquitectónico:
- Sin cambios de módulos ni patrón. Se añade vertical de localización respetando capas domain/data/presentation/composeApp.

Pendientes:

- Aplicar locale actual a toda la UI (titles, labels y contenido) conforme avance la migración de strings.
