## 2026-05-11 (PR 3 KMP/Android Billing boundary)

- Android/KMP Billing: `SubscriptionBillingDataSource` queda limitado a catálogo de productos y tokens restaurables; `GooglePlaySubscriptionBillingDataSource` devuelve `BillingProduct`, `BillingPurchaseToken` y `BillingPurchaseResult` sin convertir `PURCHASED` en Premium activo.
- Catálogo v1: se configura únicamente el producto mensual real `bwitch_premium_monthly`; anual permanece reservado y fuera del query/render de esta fase.
- Settings/analytics: una compra local exitosa o pending ya no dispara `premium_purchase_completed` ni refresca Premium desde Billing local; restore devuelve tokens para validación backend posterior y no concede Premium local.
- Impacto arquitectónico: se refuerza que `/userEntitlements/{uid}` y los callables backend son la única autoridad Premium; la cache legacy `subscription_status` solo puede leerse como estado informativo no autoritativo.
- Pendientes: conectar `purchaseToken` con `validateGooglePlaySubscription`/`restoreGooglePlayPurchases`, refrescar entitlement backend en UI, implementar UI Premium final y mantener fuera de scope moon packs/App Store/anual hasta sus PRs.

## 2026-05-11 (PR 4 Premium entitlement client wiring)

- Se añadió la capa cliente común de Premium Entitlements para invocar `validateGooglePlaySubscription`, `restoreGooglePlayPurchases` y `refreshEntitlement` desde KMP.
