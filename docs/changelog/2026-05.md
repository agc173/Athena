# Changelog — 2026-05

## 2026-05-06

- Synastry UI: responsive pass on form labels, dropdown text, metric rows, and daily energy axis headers to prevent narrow-device text overflow/orphan wrapping.
- Impacto arquitectónico: ninguno; cambio limitado a UI Compose y documentación de changelog.
- Pendientes: validar visualmente en dispositivos físicos pequeños con escalas de fuente altas y localizaciones largas.

## 2026-05-08

- Daily-limit UX: added a reusable paywall-style limit card and mapped expected daily/free/moon limit states in Birth essence, Synastry, Tarot, Oracle, and Pendulum away from red error UI; CTA copy points to the store while Premium is not implemented.
- Impacto arquitectónico: ninguno; cambio limitado a UI Compose, presentación, callbacks de navegación a MoonStore y strings localizados.
- Pendientes: validar visualmente el copy localizado en dispositivos estrechos cuando el flujo Premium definitivo esté disponible.

- Play Billing Premium sandbox: alineado el cliente Android con el producto mensual real `bwitch_premium_monthly` y base plan `monthly`; el catálogo queda mensual-only, compra/restore conservan `purchaseToken` y Premium solo se activa tras entitlement backend activo (`validateGooglePlayPurchase` / `restoreGooglePlayPurchases` / `refreshEntitlement`).

## 2026-05-12
- Backend Premium Google Play sandbox: añadidas callables `validateGooglePlayPurchase`, `restoreGooglePlayPurchases` y `refreshEntitlement` en Functions, con validaciones estrictas de producto mensual/base plan/package/token, validación server-side contra Android Publisher API y escritura backend-owned de `/userEntitlements/{uid}`.
- Se añade ownership por hash en `/purchaseTokenIndex/{hash}` para impedir activar el mismo token en otro UID.
- Documentación actualizada para schema Firestore, índices, reglas y configuración Google Play/GCP; se explicita que `refreshEntitlement` no revalida Play todavía y que acknowledge sigue siendo responsabilidad del cliente después de backend active.
- Fuera de alcance mantenido: anual, App Store, Moon packs, RTDN, trials/ofertas/descuentos.

## 2026-05-13

- Paywall analytics/UX tightening: restore Premium now emits typed clicked/completed/empty events, with completed reserved for backend-active restored entitlements only.
- Daily-limit paywall card now receives module/placement/reason context, tracks one paywall impression per stable context plus store CTA clicks, and can switch to Premium-oriented copy where previews expose a Premium benefit. Tarot falls back to selected request type context when no module preview is available.
- MoonStore copy separates Premium from the future-content section, uses real monthly Premium copy, adds restore to the Premium card, and keeps Moon packs as coming-soon placeholders without enabling real pack purchases.
- Impacto arquitectónico: ninguno; cambios limitados a eventos tipados de analytics, ViewModels existentes, UI Compose y strings localizados.
- Fuera de alcance mantenido: pricing/economy rules, packs reales de Lunas, anual, App Store, RTDN, trials/ofertas y A/B testing.
